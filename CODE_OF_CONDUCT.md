# Code of Conduct

[中文](#中文) | English

## Our pledge

We are committed to making participation in this project a respectful and harassment-free experience for everyone.

## Expected behavior

- Be respectful and constructive.
- Assume good intent when reviewing issues and pull requests.
- Keep discussions focused on the project and its users.
- Provide clear reproduction details for bugs and compatibility reports.
- Avoid sharing secrets, credentials, private keys, or sensitive production data.

## Unacceptable behavior

- Harassment, threats, insults, or personal attacks.
- Publishing another person's private information.
- Spam, trolling, or deliberately derailing technical discussions.
- Submitting malicious code, credential leaks, or intentionally unsafe examples.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other contributions that violate this code of conduct. Repeated or severe violations may result in being blocked from participating in the project.

## Reporting

If you see behavior that violates this code of conduct, contact the maintainers through the repository owner profile. For security-sensitive issues, follow `SECURITY.md` instead of opening a public issue.

# 中文

## 我们的承诺

我们致力于让每个人都能以尊重、免受骚扰的方式参与本项目。

## 期望行为

- 保持尊重和建设性。
- 在 review issue 和 pull request 时默认对方是善意的。
- 讨论聚焦项目和用户价值。
- 报告 bug 和兼容性问题时提供清晰复现信息。
- 避免分享密钥、凭据、私钥或敏感生产数据。

## 不可接受行为

- 骚扰、威胁、侮辱或人身攻击。
- 公开他人隐私信息。
- 垃圾信息、钓鱼式争论或故意带偏技术讨论。
- 提交恶意代码、泄露凭据或故意不安全的示例。

## 执行方式

维护者可以编辑、隐藏或删除违反本行为准则的评论、issue、pull request 或其他贡献。重复或严重违规者可能会被限制继续参与项目。

## 报告方式

如果你发现违反本行为准则的行为，请通过仓库 owner 个人资料联系维护者。安全敏感问题请遵循 `SECURITY.md`，不要提交公开 issue。
